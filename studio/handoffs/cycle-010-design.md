# Cycle 10 — Design

## Item
BACKLOG-049 [ai] Offload WebLLM to a Web Worker + surface brain status & reply source.

## Why this cycle
The model runs on the main thread → the game stutters while a dino thinks (operator-observed). And the dialog reads as "mostly hellos" with no way to know if a line is from the LLM or the canned fallback. This cycle moves inference into a Web Worker (smooth gameplay) and makes the mind legible: a brain-status readout and a per-reply source tag.

## What ships
WebLLM loads and runs in a dedicated Web Worker via `CreateWebWorkerMLCEngine`, so the model download and token generation no longer block the render loop. A small status line (top-right) shows the brain state: `🧠 zzz` (idle), `🧠 thinking…` (loading), `🧠 ready`, or `🧠 offline` (fallback / no WebGPU). Each reply now carries a `source` (`'llm'` | `'canned'`); when a dino's line was actually generated by the model, the dialog box prefixes it with `🧠`. So the player can tell a real line from the safety net at a glance.

Dev hooks: existing `__brainStatus()` stays; add `__lastReplySource()` → `'llm' | 'canned' | null`.

## Acceptance criteria
- [ ] `Reply` carries a `source` field; `cannedReply` returns `source: 'canned'`; a successful model generation returns `source: 'llm'` (unit, via fake engine).
- [ ] When the model isn't ready, `respond()` returns `source: 'canned'` (unit).
- [ ] The brain-status HUD text reflects `__brainStatus()` and updates as status changes (e2e: status string is one of the known states; HUD exists).
- [ ] The dialog prefixes generated lines with 🧠 and leaves canned lines unprefixed (unit on the prefix helper + e2e that the helper is wired).
- [ ] The worker engine is created via a real `Worker` only in the browser; Node unit tests still use the injected fake loader and never spawn a worker (unit).
- [ ] WebLLM remains confined to `game/src/ai/` (now `webllmBrain.ts` + `webllm.worker.ts`, both under `ai/`) — no scene/entity imports it (grep).
- [ ] No regression: clock/day-night/save/roster/hearts/gifts/brain-fallback suites green (e2e).
- [ ] `npm run build` clean (Vite bundles the worker); vitest + playwright green.
- [ ] **Human (manual):** on a WebGPU browser, greeting shows `🧠 thinking…` → `🧠 ready` with no gameplay freeze, and generated lines carry the 🧠 tag.

## Out of scope
- Token streaming / partial display.
- Changing the prompt or `cleanReply` (cycle 8 owns voice; dialogue variety is a later item).
- Model-size selection (006).

## Constraints
- Worker file lives in `game/src/ai/` — boundary holds.
- The injected-loader test seam (cycle 7) is preserved: tests never construct a real `Worker`.
- `source` is additive on `Reply` (optional) — must not break existing brain tests.
- No new dependencies (web-llm already provides the worker engine). TS strict; `any` only via documented patterns.
- Status HUD + dialog prefix built in `WorldScene` like the other HUDs.
